## Sobre esta organización

Esta organización de GitHub fue creada para centralizar el trabajo de la materia **Informática Aplicada II**.

Acá cada persona va a contar con su propio repositorio para desarrollar, versionar y entregar sus trabajos prácticos. La idea es que todo el proceso quede ordenado, trazable y accesible: código, documentación, avances, correcciones y entregas finales.

Además de programar o producir el material solicitado, también es importante que cada persona mantenga su repositorio prolijo, documentado y actualizado.

---

## ¿Cómo se organiza el trabajo?

- Cada alumnx tendrá su propio repositorio. Para ello, deberá tener usuario en Github y los docentes le crearemos el repositorio con los permisos correspondientes.
- El nombre del repositorio será el **nombre y apellido** de la persona.
- Cada alumnx deberá subir allí sus **trabajos prácticos**, avances y versión final de entrega. En caso de tener que recuperar, también se deberá subir el trabajo al repositorio con el nombre correspondiente.

---

## Modalidad de entrega

Cada entrega deberá realizarse dentro del repositorio asignado al equipo en fecha y hora determinados por el EvaED. Allí tendrán que cargar los links formalmente hasta antes de la fecha y hora de cierre.

Se espera que en cada repositorio estén, como mínimo:

- El código fuente o archivos del trabajo.
- Un `README.md` con una breve explicación del trabajo realizado.
- Instrucciones para ejecutar o abrir el trabajo.
- Los archivos complementarios necesarios para corregir la entrega.
- Una organización clara por carpetas, en caso de haber múltiples entregas o versiones.

---

## 🏗️ Estructura del repositorio

```bash
apellido-nombre/
│
├── README.md
├── apellido_nombre_entrega_tp1
├── apellido_nombre_recuperatorio_tp1 # en caso de recuperar
├── apellido_nombre_entrega_tp2
├── apellido_nombre_recuperatorio_tp2 # en caso de recuperar
├── apellido_nombre_entrega_tp3
├── apellido_nombre_recuperatorio_tp3 # en caso de recuperar



```
Nota: recordar siempre subir las carpetas de cada TP en la raíz del repositorio y evitar subir carpetas anidadas, es decir, una dentro de otra. Respetar la nomenclatura de las carpetas, y que haya una correspondiente a la preentrega, entrega y recuperatorio de cada TP.
